Atlas v1 Windows 11 Python 3.12

Atlas 部署與使用說明書

Atlas 是一個 Python terminal 工具，用系統 Chrome 操作公司內部 tGenie，讓 tGenie 可以在單一 workspace 內讀檔、寫檔、上傳 PDF 或圖片附件、產出 LLM Wiki 與 HTML 檢視。

1. 這份手冊適用範圍

正式入口
atlas terminal TUI。
主要環境
Windows 11、Python 3.12.x、Google Chrome。
主要輸出
TUI 回覆、workspace 檔案、LLM Wiki Markdown、HTML、graph。

2. 部署需求

  • Windows 11。
  • Python 3.12.x。
  • Google Chrome。
  • 可開啟 tGenie 的公司網路與 tGenie URL。
  • PowerShell。
py -3.12 --version
Atlas 使用系統 Chrome，不使用 Playwright 下載的瀏覽器。tGenie 登入由使用者在 Chrome 裡手動完成，Atlas 不會處理密碼或 SSO 憑證。

3. Windows 部署步驟

在 Atlas 專案資料夾執行：

py -3.12 -m venv .venv
.\.venv\Scripts\python -m pip install -e .
.\.venv\Scripts\atlas --help

啟動 Atlas：

.\.venv\Scripts\atlas

指定 workspace：

.\.venv\Scripts\atlas C:\path\to\workspace

從任意 PowerShell 執行

$atlasScripts = "$PWD\.venv\Scripts"
$userPath = [Environment]::GetEnvironmentVariable("Path", "User")
[Environment]::SetEnvironmentVariable("Path", "$atlasScripts;$userPath", "User")

關掉 PowerShell 並重新開啟後，可以執行：

atlas
atlas C:\path\to\workspace

6. Input to Output Workflow

一般對話

Input使用者在 Atlas TUI 輸入 prompt。
ProcessAtlas 送到 tGenie。tGenie 直接回答，或用 atlas.tool_call 要求本機工具。Atlas 執行工具後回貼 atlas.tool_result。
OutputTUI 顯示 Working: 狀態與最後 Atlas: 回覆。若使用 file.write，workspace 會產生或更新檔案。

做法：tGenie 負責推理，Atlas 負責本機工具。

原因：網頁版 tGenie 不一定有 native function calling，文字型 tool-call protocol 可以先完成本機 agent workflow。

影響與取捨：安全邊界清楚；一次只處理一個 tool call，速度較慢但較穩。

Workspace 檔案工作

Input要求讀檔、搜尋、整理或產出檔案。
ProcessAtlas 只允許 file.list、file.read、file.search、file.write 操作 workspace 內 path。
Output回答顯示在 TUI；檔案輸出寫在 workspace 內。

PDF 摘要或分析與圖片附件

Input例如「請摘要 docs/example.pdf」或「請看 photos/panel.png」。
ProcesstGenie 要求 file.attach，Atlas 驗證 workspace-local .pdf、.jpg、.jpeg 或 .png，然後透過 tGenie 網頁上傳。
OutputtGenie 讀取 PDF 或圖片後，Atlas TUI 顯示摘要或分析。

LLM Wiki 匯入

/llm-wiki ingest docs/example.pdf
/llm-wiki ingest docs/pdf-folder
Input單一 PDF 或 workspace 內 PDF 資料夾。
ProcessAtlas 初始化 wiki/，每批 1 個 PDF 送給 tGenie，讓 tGenie 更新 wiki Markdown，最後重新產生 HTML mirror 與 graph。
Outputwiki/index.md、wiki/log.md、wiki/pages/、wiki/output/html/index.html、wiki/output/graph/index.html。

Skill 使用

Input/llm-wiki 或 /skill-creator。
ProcessAtlas 載入內建或 workspace-local skill instructions。
OutputTUI 顯示 skill 已載入；後續 prompt 依該 skill 工作。

8. Workspace 安全規則

  • Atlas 啟動時只綁定一個 workspace。
  • 檔案工具都只能操作 workspace 內 path。
  • file.attach 只接受 workspace 內 .pdf、.jpg、.jpeg、.png。
  • pdf.attach 保留給舊流程，只接受 workspace 內 .pdf。
  • absolute path、.. escape、指向 workspace 外的 path 會被拒絕。
  • shell.run 預設保守，大部分 command 會回傳 confirmation-required，高風險 command 會回傳 rejected。

9. 常見問題

atlas 找不到

如果尚未設定 PATH，請使用：

.\.venv\Scripts\atlas

Chrome 沒有開啟

確認公司電腦已安裝 Google Chrome，且公司政策允許本機 Python 程式開啟 Chrome。

Atlas 一直要求 /login-done

先在 Chrome 裡完成 tGenie 登入，再回到 Atlas TUI 輸入 /login-done。

附件上傳失敗

  • 確認檔案在 workspace 內。
  • 確認副檔名是 .pdf、.jpg、.jpeg 或 .png。
  • 確認 path 沒有使用 absolute path 或 ..。
  • 確認附件不含不該上傳的敏感資料。
