公司電腦操作步驟

1. 取得新版程式

如果公司電腦沒有 git，就從 GitHub 下載最新 ZIP，解壓後進入 Atlas 資料夾。

py -3.12 -m venv .venv
.\.venv\Scripts\python -m pip install -e .
.\.venv\Scripts\python scripts\probe_tgenie.py --url "https://your-company-tgenie-url"

2. 進入 tGenie 對話畫面

登入後停在 tGenie 主畫面。你不需要重新記錄既有 selector，只要能建立新對話並送出 smoke prompt。

如果畫面狀態和昨天一樣，可以照既有資訊操作：

  • 點 sidebar toggle。
  • 點 New Conversation。
  • 確認模型是 Gemini-3.1-Pro Preview。
  • 送出 smoke prompt。

3. 在 probe 裡只補 latest response

送出 smoke prompt 並看到 tGenie 回覆 atlas-ok 後，在 probe terminal 輸入：

texts

在輸出中找到包含 atlas-ok 的文字區塊編號，例如 [37]。接著輸入：

set_text latest_response 37
note latest_response_text atlas-ok
note latest_response_rule bottom-most assistant response text block after the smoke prompt
note smoke_result success
done
注意： 上面的 37 是例子。明天要用 texts 實際列出的編號。

回家後只要帶回這些文字

不用帶圖。只要從 probe-output/tgenie_probe_*.md 抄下面這些非敏感段落：

## 校準結果
- **latest_response** ...

## #5 必填觀察
- **latest_response_rule** ...
- **latest_response_text** ...
- **smoke_result** ...

如果 texts 找不到 atlas-ok

  1. 先輸入 wait 3000，再輸入 texts。
  2. 如果還是沒有，手動複製 tGenie 回覆文字附近的非敏感文字。
  3. 記錄：note latest_response_text atlas-ok。
  4. 記錄：note latest_response_rule texts did not expose the response; visible final response was atlas-ok。
